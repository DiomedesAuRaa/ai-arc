================================================================================
AI AGENT ALLOWED COMMANDS — NON-DESTRUCTIVE OPERATIONS
================================================================================
This file defines the exhaustive list of commands that AI agents (Claude, Cursor,
Copilot) are permitted to suggest and execute autonomously. All commands must be
non-destructive: they must not change configuration, delete resources, or apply
writes against production data. Commands may update local tool state (e.g.,
Terraform plans, kubeconfig updates) and may generate logs/metrics on remote
systems, but must not perform irreversible or destructive actions.

Agents MUST NOT execute any command not listed here without explicit human
confirmation. When in doubt, propose the command and wait for approval.
================================================================================

================================================================================
RULES FOR AI AGENTS
================================================================================
1. Only suggest or execute commands from the above list.
2. Never chain a read command into a write (e.g., no `> file`, `>> file`).
3. Never pipe into any command not on this allow list.
4. Never use sudo unless the command is explicitly non-destructive and the agent
   has been granted explicit human approval for that specific invocation.
5. For `gh api`: only construct GET requests. Never infer POST/PUT/PATCH/DELETE.
6. For `curl`: always require explicit `-X GET` or `--head`/`-I`. Reject any
   curl invocation without an explicit method — bare `curl <url>` with a body
   defaults to POST.
7. For `sed`: only allow the pattern `sed -n '<addr>p'` or `sed -n '<addr>s/.../p'`.
   The `-i` flag is never allowed under any circumstances. If the intent can be
   achieved with `grep` or `awk`, prefer those — they have no write mode at all.
8. For `find`: only the explicit patterns listed above are permitted. The flags
   `-delete`, `-exec`, `-execdir`, `-ok`, and `-okdir` are never allowed, even
   when the downstream command appears safe. `find` must not be used as a
   command compositor.
9. `xargs` is not on the allow list and must never be used. It is a command
   execution primitive that cannot be safely constrained to read-only via
   prompt instructions alone.
10. `tee` is not on the allow list and must never be used. `tee` always writes
    to a file descriptor; there is no read-only invocation.
11. For `aws`, `gcloud`, `az`: if a subcommand is not explicitly listed above,
    treat it as unapproved and ask for human confirmation before executing.
12. Secret values should not be printed, logged, or surfaced unless the human
    explicitly requests them for a specific debugging or integration task. Even
    then, they must never be copied into long-lived logs or external systems.
13. When uncertain whether a command or flag combination is non-destructive, do
    not execute — propose it and wait for explicit human confirmation.
================================================================================
